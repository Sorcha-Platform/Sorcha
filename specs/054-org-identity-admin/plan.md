# Implementation Plan: Organization Admin & Identity Management

**Branch**: `054-org-identity-admin` | **Date**: 2026-03-08 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/054-org-identity-admin/spec.md`

## Summary

Implement multi-tenant organization administration with OIDC-based external identity provider integration, social login for public organizations, role consolidation (8в†’5 roles), email verification, invitation-based onboarding, domain restrictions, multi-tier URL resolution, admin console with user management, and audit logging. Builds on the existing Tenant Service which already has Organization/UserIdentity models, JWT token issuance, TOTP 2FA, and audit infrastructure.

## Technical Context

**Language/Version**: C# 13 / .NET 10
**Primary Dependencies**: ASP.NET Core Minimal APIs, Entity Framework Core (PostgreSQL), Redis, MailKit (email), Microsoft.IdentityModel.Tokens (JWT/OIDC validation), Blazor WASM (admin UI)
**Storage**: PostgreSQL (Tenant Service вЂ” `sorcha_tenant` database, multi-tenant schema isolation), Redis (token revocation, session cache, custom domain cache)
**Testing**: xUnit + FluentAssertions + Moq, WebApplicationFactory for integration tests
**Target Platform**: Linux containers (Docker/Azure Container Apps)
**Project Type**: Microservice extension (Tenant Service) + Admin UI (Blazor WASM)
**Performance Goals**: OIDC callback < 2s, URL resolution < 500ms, dashboard load < 1s
**Constraints**: No new service projects вЂ” all backend work in Tenant Service. Full token exchange (external tokens never leave Tenant Service).
**Scale/Scope**: ~30 new endpoints, ~6 new/modified entities, ~15 new service classes, ~200 new tests

## Implementation Phases (High-Level)

### Phase 1: Data Model & Role Consolidation
- EF Core migration: modify Organization, UserIdentity, IdentityProviderConfiguration
- Add new entities: OrgInvitation, CustomDomainMapping
- Role consolidation migration (8в†’5): map Developer/User/Consumer в†’ Member
- Update UserRole enum, authorization policies, token claims
- **Tests**: Migration tests, role mapping tests

### Phase 2: OIDC Discovery & IDP Configuration
- OidcDiscoveryService (fetch + cache `.well-known/openid-configuration`)
- IdpConfigurationService (CRUD, discover, test, toggle)
- IDP configuration endpoints
- Provider presets (top 5 shortcuts)
- **Tests**: Discovery service tests, IDP config CRUD tests

### Phase 3: OIDC Authentication Flow
- OidcExchangeService (authorization URL generation, code exchange, token validation)
- OidcProvisioningService (claim mapping, auto-provisioning, domain restriction check)
- OIDC endpoints (initiate, callback, profile completion)
- Email verification flow (token generation, verification endpoint)
- **Tests**: Full OIDC flow integration tests with mocked IDP

### Phase 4: Local Auth Enhancements
- PasswordPolicyService (NIST policy, HIBP breach check)
- Self-registration endpoint (public orgs)
- Email sending abstraction (IEmailSender + SmtpEmailSender)
- Progressive lockout verification (confirm thresholds match spec)
- **Tests**: Password policy tests, registration flow tests

### Phase 5: Invitation System & Domain Restrictions
- InvitationService (create, accept, revoke, expire)
- Invitation endpoints
- Domain restriction endpoints
- Invitation email sending
- **Tests**: Invitation lifecycle tests, domain restriction tests

### Phase 6: Custom Domain & URL Resolution
- CustomDomainService (configure, verify CNAME, status tracking)
- CustomDomainMapping repository
- Custom domain endpoints
- Background verification service (daily CNAME check)
- API Gateway path-based URL resolution middleware
- **Tests**: Domain verification tests, URL resolution tests

### Phase 7: Admin Console UI
- Dashboard page (stats, IDP status, recent logins)
- User management page (search, filter, role change, suspend/activate)
- IDP configuration page (wizard with provider shortcuts)
- Invitation management page
- Domain restrictions page
- Audit log viewer page
- Organization settings page
- **Tests**: Blazor component tests, E2E Playwright tests

### Phase 8: Audit & Operational
- Extend AuditEventType enum with new event types
- Wire audit events into all new services
- AuditCleanupService (background retention purge)
- Audit query endpoint with filtering/pagination
- Dashboard statistics endpoint
- Update YARP routes for new endpoints
- **Tests**: Audit logging tests, cleanup service tests

### Phase 9: Integration & Documentation
- End-to-end integration tests (full OIDC flow)
- Update API documentation
- Update MASTER-TASKS.md
- Update service README
- API Gateway route additions
- Walkthrough creation
