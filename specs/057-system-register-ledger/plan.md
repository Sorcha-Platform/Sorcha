# Implementation Plan: System Register as Real Ledger

**Branch**: `057-system-register-ledger` | **Date**: 2026-03-15 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/057-system-register-ledger/spec.md`

## Summary

Replace the separate MongoDB-backed system register with a real register that uses the standard ledger infrastructure. The `SystemRegisterBootstrapper` will use the existing `RegisterCreationOrchestrator` two-phase flow to create the system register on first startup, then submit seed blueprints as transactions via `IValidatorServiceClient`. The `SystemRegisterService` becomes a thin query layer over register transactions. The old `MongoSystemRegisterRepository` and `SystemRegisterEntry` are removed entirely.

## Technical Context

**Language/Version**: C# 13 / .NET 10

**Performance Goals**: Bootstrap completes within 60 seconds of first startup
**Constraints**: Idempotent bootstrap, deterministic register ID, backward-compatible API surface
**Scale/Scope**: Modifies Register Service, Validator Service (config), UI Core, removes ~3 files, refactors ~5 files
