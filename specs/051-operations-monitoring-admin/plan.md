# Implementation Plan: Operations & Monitoring Admin

**Branch**: `051-operations-monitoring-admin` | **Date**: 2026-03-06 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/051-operations-monitoring-admin/spec.md`

## Summary

Wire existing backend endpoints to Blazor WASM UI components and CLI commands for 6 operational admin features (dashboard alerts, wallet delegation, schema providers, events admin, push notifications, encryption progress), plus code quality improvements from the Feature 050 PR review (credential status constants, typed error handling, structured multi-value inputs, presentation request auto-refresh, shared JSON config).

All backend endpoints already exist вЂ” this feature is exclusively UI/CLI wiring, new UI pages/components, and service reliability improvements. No backend development required.

## Technical Context

**Language/Version**: C# 13 / .NET 10
**Primary Dependencies**: Blazor WASM (MudBlazor), System.CommandLine 2.0.2, Refit, Spectre.Console
**Storage**: Browser localStorage (alert dismissals only вЂ” no new backend storage)
**Testing**: xUnit + FluentAssertions + Moq (unit), Playwright (E2E)
**Target Platform**: Blazor WASM (browser) + CLI (.NET console)
**Project Type**: Web (Blazor WASM SPA + CLI tool)
**Performance Goals**: Dashboard refresh every 30s, presentation polling every 5s, encryption progress within 1s of stage transition
**Constraints**: Follow existing `AuthenticatedHttpMessageHandler` pattern for new services; per-user alert dismissal via localStorage
**Scale/Scope**: ~15 new files, ~10 modified files, ~250 new tests targeting >85% coverage

## Implementation Phases

### Phase 1: Foundation & Code Quality (FR-024, FR-025, FR-028)
1. Create `CredentialStatus` constants class
2. Replace all credential status magic strings across UI components
3. Add typed error returns to `CredentialApiService` lifecycle methods
4. Update `CredentialLifecycleDialog` to display specific error messages
5. Extract shared `JsonSerializerOptions` configuration
6. Tests for all changes

### Phase 2: Dashboard & Alerts (FR-001 to FR-004)
1. Add 30-second auto-refresh timer to `Home.razor`
2. Integrate `AlertsPanel` component into Home page
3. Implement per-user alert dismissal (localStorage)
4. Handle graceful "unavailable" state when stats endpoint unreachable
5. Tests: unit for alert service, E2E for dashboard

### Phase 3: Wallet Access Delegation (FR-005 to FR-009)
1. Create `IWalletAccessService` + `WalletAccessService`
2. Create `WalletAccessTab.razor` component
3. Add Access tab to `WalletDetail.razor`
4. Add `wallet access grant|list|revoke|check` CLI commands
5. Register service in `ServiceCollectionExtensions`
6. Tests: unit for service, unit for CLI commands, E2E for tab

### Phase 4: Events Admin (FR-014 to FR-017)
1. Create `IEventAdminService` + `EventAdminService`
2. Create `EventsAdmin.razor` page with filtering and pagination
3. Add `admin events list|delete` CLI commands
4. Register service and add nav menu entry
5. Tests: unit for service, unit for CLI, E2E for page

### Phase 5: Push Notifications & Schema CLI (FR-013, FR-018 to FR-020)
1. Create `IPushNotificationService` + `PushNotificationService`
2. Create `NotificationSettings.razor` page with toggle
3. Handle browser permission denial gracefully
4. Add `schema providers list|refresh` CLI commands
5. Tests: unit for service, unit for CLI, E2E for settings

### Phase 6: Encryption Progress & Structured Inputs (FR-021 to FR-023, FR-026, FR-027)
1. Create `IOperationStatusService` + `OperationStatusService`
2. Create `EncryptionProgressIndicator.razor` component
3. Add `operation status` CLI command
4. Replace comma-separated inputs with tag/chip inputs on PresentationAdmin
5. Add 5-second auto-refresh to PresentationAdmin for pending requests
6. Tests: unit for service, unit for CLI

### Phase 7: E2E Tests & Integration Verification
1. Complete any remaining E2E tests
2. Verify all new nav entries and routing
3. Cross-feature integration smoke test
4. Documentation updates (README, MASTER-TASKS)
